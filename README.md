## サービス概要
  欲しいもの・買うか迷うものに対して「なぜいま買うのか？」を考える時間を作ることで、納得した購入判断をサポートするWebアプリです。質問への回答やジャーナリング(書く瞑想)、クールダウンタイマーを通して、衝動的な購入を防ぎながら、思考や感情を整理することができます。

## 開発の背景
  私は以前、書籍や楽譜を買いすぎてしまったことがあります。一点一点は比較的安価だったため金額によるブレーキが効きづらく、まとめ買いを繰り返してしまいました。その結果、「本当にすべて手を付けられるのだろうか」という物を持ちすぎていることへの負担感や可処分時間への不安を強く感じるようになりました。

  振り返ると、私の場合は家計簿をつけるだけでは買いすぎを防ぐことができず、「本当にいま買う必要があるのか」「ストレスによる衝動買いではないか」などを冷静に考え、自分の状態や思考のクセに気付くことが必要だったと思いました。

  一方で、節約だけを重視すると、学び・経験・交流など本来使うべき場面でお金を使えないこともあるのではないかと考えました。

  そこで「買わない」ことではなく、購入判断をサポートすることで「後悔を減らす」ことを目的にしたサービスを作ることにしました。

## 想定ユーザー
  - 衝動買い・無駄遣いを減らしたいと思っている方
  - 家計簿や節約系アプリでは衝動買い・無駄遣いを防げなかった方
  - 購入前に一度立ち止まって考える習慣を作りたい方
  - きちんと考えて買い物をしたいけど、何から取り組めば良いか分からない方

## サービスの利用イメージ
  1. 欲しいもの(買うか迷うもの)を登録する
  2. 質問に回答し、欲しい理由や現在の気分を整理する
  3. クールダウンタイマーを設定し、購入判断を保留する
  4. タイマー終了後に通知を受け取る
  5. ジャーナリングを行い、自分の気持ちや考えを整理する
  6. 「買う」「買わない」を改めて判断する
  7. 判断結果を蓄積し、自分の思考パターンを振り返る

## サービスの特徴（既存サービス・競合調査）
  既存の衝動買い・無駄遣い防止サービスは、節約や支出管理を目的にしたものが多いです。
  - 節約をサポートするもの(例：家計簿、最安値を見つける)
  - 金額を見える化するもの(例：無駄遣い金額や節約金額の可視化)
  - 買わないことを目指すもの(例：我慢した回数の記録、AIが購入の必要性を問いかける)

  一方、本サービスでは「なぜそれを欲しいと思ったのか」という思考整理と感情認識に焦点を当てています。感情を言語化・数値化すること(ラベリング)は衝動を抑え、ジャーナリングは自己理解の促進やストレス軽減に効果があるとされています。サービス利用の流れの中で、

  - 質問による感情のラベリング
  - クールダウンタイマー
  - ジャーナリング

  を組み合わせることで、衝動を落ち着かせた状態で購入判断ができる仕組みを提供します。<br>
  また、「買う」「買わない」どちらの判断も尊重し、ユーザーが納得して意思決定できる体験を大切にしています。

## 機能候補
  ### MVPリリース
  - ユーザー登録・ログイン
  - 欲しいもの(買うか迷うもの)登録
    - どのくらい欲しいか・今の気分(5段階で回答)
    - なぜ欲しいかを考えるきっかけになる質問(選択式)
    - ジャーナリング(書く瞑想)
    - 購入判断(買う・買わない)
  - クールダウンタイマー(一定時間経過後にメール通知)
    - 1回目：「30分」「24時間」「3日」から選択 or スキッ
    プ
  - 判断履歴の表示

  ### 本リリース
  - ソーシャルログイン(OmniAuth)
  - 欲しいもの(買うか迷うもの)登録の拡張
    - クイック登録(欲しいもの名だけ先に登録するための動線)
    - 質問内容のカスタマイズ(ユーザーが変更可能にする)
    - カテゴリタグ(判断履歴の表示・購入傾向の分析に活用)
  - 通知機能の拡張(LINE Messaging API)
  - 登録済み情報の検索
  - 購入後の満足度登録(3段階で回答)
  - 購入傾向の分析(登録データを活用したグラフ表示)
  - 気づき・目標の登録(欲しいもの単体ではなく、購入判断全体に対するもの)

  ### 懸念点と対策
  #### ユーザーが入力や回答を面倒に感じ、継続できない可能性

  ジャーナリングは継続することで効果が出やすいとされていますが、毎回文章を書くことが負担になる可能性があります。

  **対策：**
  - 質問回答をスキップ可能にする
  - 欲しいもの名だけ先に登録できるクイック登録機能を用意する
  - 短時間で回答できるよう、選択式の質問形式を採用する
  - 時間がある時に落ち着いてジャーナリングを行える設計にする
  - ユーザーの状況に応じて負担を調整できるようにする

  #### サービスの価値がユーザーに伝わりにくい可能性

  すぐに効果が実感できるサービスではないため、利用価値が伝わりにくい可能性があります。

  **対策：**
  - ユーザーのフィードバックを元に、サービス利用方法の伝え方や機能面を改善する
  - 購入傾向の分析機能を用意することで、ユーザーがより良い購入判断に繋げられるようにする

  ### 今後の展開・発展の方向性
  ジャーナリングの継続や、購入判断データの蓄積によって、以下のような価値を提供できると考えています。
  - ユーザーの自己理解の促進
  - 衝動買いしやすい状況や、思考・行動パターンの可視化

  #### 選定理由
  短期間でMVPを開発し、ユーザーの反応をもとに改善を行うことを目的としているため、使用経験のある技術を中心に選定しました。Railsの標準機能やGemによって必要な機能を効率よく実装できると考えています。

  #### チャレンジしたい点・不安な点
  - Hotwire(Turbo/Stimulus)を活用した動的UIの実装
  - LINE通知機能の実装
  - データを活用した分析機能の実装
  - デプロイを開発初期に行うことでリスクを軽減
  - 通知機能や外部APIの実装経験が少なく、開発途中で課題が発生する可能性があるため、段階的な機能追加を意識
